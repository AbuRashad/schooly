وصف الاختراع – طبيعة البرنامج والمشكلة التي يحلها

Schooly هو نظام ذكي متكامل لمراقبة وتأمين بيئات المدارس في الوقت الفعلي، يجمع بين تقنيات الذكاء الاصطناعي ومعالجة الفيديو وتحليل البيانات السلوكية داخل منصة موحدة ومتعددة المستويات مصممة خصيصاً للمؤسسات التعليمية العربية.


المشكلة: تفتقر المدارس إلى أنظمة مراقبة ذكية قادرة على الربط الآني بين كاميرات المراقبة وبيانات الحضور وسلوك الطلاب وتنبيه الإدارة فور وقوع حوادث أمنية أو سلوكية، مع ضمان خصوصية البيانات وفق متطلبات حوكمة البيانات العربية.

الحل: توفر Schooly منظومة من 15 وحدة ذكاء اصطناعي تعمل بالتوازي لتحقيق مراقبة شاملة تغطي الدخول والممرات والفصول والملاعب، مع توليد تقارير دورية متعددة المستويات وبوابة أولياء أمور تتيح تتبع الطالب في الوقت الفعلي.
٢

وظائف البرنامج الرئيسية

المراقبة الآنية

استقبال بث RTSP من كاميرات المراقبة الموزعة، تحليل المشاهد، ورصد الكثافة والحركة في جميع مناطق المدرسة.

الحضور الذكي

تسجيل حضور الطلاب تلقائياً عبر التعرف على الوجه بدون تدخل بشري، مع دمج بيانات الحضور وأحداث السلامة.

الإنذار المبكر

كشف السلوكيات الخطرة وإرسال تنبيهات فورية للإدارة مصنفة حسب مستوى الخطورة مع توصيات الاستجابة.

التقارير متعددة المستويات

توليد تقارير تشغيلية وتحليلية وإشرافية ووزارية بصيغ متنوعة لدعم القرار على جميع المستويات.

بوابة أولياء الأمور

واجهة آمنة تتيح لأولياء الأمور متابعة موقع الطالب وحضوره وسجل السلامة الخاص به في الوقت الفعلي.

الذكاء البيداغوجي

تحليل سلوك الطلاب التعليمي وقياس مستوى التفاعل والانتباه داخل الفصل وتقديم توصيات تربوية.

وصف الوحدات الذكية الخمس عشرة

رقم اسم الوحدة الوظيفة والمخرجات
1 وحدة التقاط الفيديو
Video Capture Unit		 استقبال ومعالجة بث RTSP من شبكة الكاميرات الموزعة، ضمان استمرارية البث وإدارة التدفق.
2 وحدة فهم المشهد
Scene Understanding		 تحليل إطارات الفيديو باستخدام رؤية الحاسوب (OpenCV) للكشف عن الأشخاص والأشياء والأحداث.
3 وحدة تتبع المسار
Path Tracking		 تتبع مسار حركة الطلاب والأشخاص عبر مناطق المدرسة المختلفة وبناء خرائط الحركة.
4 وحدة كشف المؤشرات الخطرة
Hazard Indicator Detection		 رصد السلوكيات والمواقف الخطرة (العنف، التجمعات المشبوهة، الحوادث) وإطلاق تنبيهات فورية.
5 الذاكرة السلوكية المكانية
Spatial Behavioral Memory		 حفظ وتحليل الأنماط السلوكية المرتبطة بمناطق محددة بمرور الوقت لبناء خريطة السلوك المكانية.
6 وحدة التماسك الجماعي
Collective Behavioral Coherence		 تحليل ديناميكيات التجمعات والحشود لاكتشاف الاضطرابات الجماعية أو التجمعات غير الطبيعية.
7 وحدة الكثافة التنبؤية
Predictive Density Unit		 التنبؤ بكثافة الحشود ونقاط الاختناق في المستقبل القريب لتمكين التدخل الاستباقي.
8 وحدة تكامل الحضور والسلامة
Attendance-Safety Integration		 ربط سجلات الحضور المولدة آلياً مع أحداث السلامة للحصول على صورة متكاملة لكل طالب.
9 وحدة التنبيه والاستجابة
Alert & Response Unit		 توليد وتوجيه التنبيهات الفورية للجهات المعنية مع توصيات الاستجابة المصنفة حسب الخطورة.
10 لوحة المراقبة الذكية
Smart Monitoring Dashboard		 عرض مباشر للتحليلات والحوادث والخرائط الحرارية على واجهة إدارية تفاعلية.
11 وحدة التقارير الدورية متعددة المستويات
Multi-Level Periodic Reports		 توليد تلقائي لتقارير الامتثال والأداء على مستويات: تشغيلي، تحليلي، إشرافي، وزاري.
12 بوابة الوالدين الذكية
Smart Parent Portal		 واجهة مخصصة لأولياء الأمور لمتابعة الطالب وحضوره وتقارير السلامة في الوقت الفعلي.
13 وحدة التقييم الذاتي المؤسسي
Institutional Self-Assessment		 حساب مؤشر سلامة المدرسة (SSI) من 5 مصادر بيانات مستقلة لدعم قرارات التحسين المؤسسي.
14 طبقة حوكمة البيانات العربية
Arab Data Governance Layer		 حماية الخصوصية وإخفاء هوية الوجه آلياً، ضمان الامتثال لمتطلبات حوكمة البيانات العربية.
15 عميل الذكاء البيداغوجي
Pedagogical Behavioral Intelligence Agent		 تحليل تفاعل الطلاب التعليمي وقياس الانتباه ومستوى المشاركة داخل الفصل مع تقديم رؤى تربوية.

لغات البرمجة والأطر المستخدمة

المكوّن التقنية / اللغة الوظيفة في المنصة
الواجهة الأمامية
Frontend		 React 19 TypeScript 5 Vite 6 بناء الواجهة التفاعلية الديناميكية مع دعم كامل للعربية RTL وتحديث فوري للبيانات.
التصميم والمكونات
UI / Styling		 Tailwind CSS 3 shadcn/ui Framer Motion نظام تصميم موحد بمكونات جاهزة (40+ مكوّن) مع انتقالات سلسة وتصميم متجاوب.
إدارة الحالة والبيانات Zustand TanStack Query إدارة الحالة العامة للتطبيق وطلبات API مع تخزين مؤقت ومزامنة تلقائية.
خادم API (Node.js) Express 5 TypeScript Better Auth خادم API خفيف لمسارات المصادقة وإدارة الجلسات وواجهة قاعدة البيانات.
خادم الذكاء الاصطناعي (Python) FastAPI Python 3.11 Uvicorn خادم API لامتزامن عالي الأداء لتشغيل وحدات الذكاء الاصطناعي الـ15 ومعالجة الفيديو.
رؤية الحاسوب
Computer Vision		 OpenCV NumPy face-api.js معالجة بث الفيديو، الكشف عن الأشخاص والوجوه، تتبع الحركة وتحليل الكثافة.
قاعدة البيانات MySQL 3.22+ Drizzle ORM تخزين بيانات الطلاب والمدارس والحوادث وسجلات الحضور مع استعلامات آمنة من نوع TypeSafe.
النشر والبنية التحتية Docker Docker Compose Nginx نشر المنصة كحاويات منعزلة قابلة للتوسع مع توجيه الطلبات عبر Nginx كخادم عكسي.
الاتصال الفوري WebSocket MJPEG Stream بث مباشر للكاميرات والتحديثات الآنية للوحة التحكم دون الحاجة لتحديث الصفحة.

كيفية استخدام البرنامج – دليل التشغيل

١. الإعداد الأولي
  • تثبيت Docker وDocker Compose
  • نسخ ملف env.example إلى .env وتعبئة المتغيرات
  • تشغيل: docker compose up --build
  • الوصول عبر المتصفح على المنفذ 80
٢. إضافة الكاميرات
  • الدخول لصفحة الإعدادات من القائمة
  • إضافة عنوان RTSP لكل كاميرا
  • تحديد المنطقة المرتبطة (مدخل / فصل / ممر)
  • تشغيل البث والتحقق من الاتصال
٣. إدارة الطلاب
  • استيراد بيانات الطلاب من ملف CSV
  • تسجيل صور الوجه لكل طالب للتعرف
  • ربط الطالب بالفصل والصف الدراسي
  • إدخال بيانات ولي الأمر للإشعارات
٤. تشغيل الوحدات الذكية
  • الوحدات تعمل تلقائياً عند بدء النظام
  • لوحة التحكم تعرض البيانات الآنية
  • التنبيهات ترسل تلقائياً عند اكتشاف حدث
  • التقارير الدورية تولد تلقائياً

مطالبات الاختراع – ما يُطالب بحمايته

  1. منظومة وحدات الذكاء الاصطناعي الخمس عشرة المتكاملة للمراقبة المدرسية في الوقت الفعلي.
  2. طريقة تكامل الحضور بالتعرف على الوجه مع بيانات السلامة الأمنية في نظام موحد.
  3. آلية توليد التقارير الآلية متعددة المستويات (تشغيلي – تحليلي – إشرافي – وزاري).
  4. طبقة حوكمة البيانات العربية مع آلية إخفاء هوية الوجه بشكل آلي وانتقائي.
  5. الذكاء البيداغوجي لقياس تفاعل الطلاب داخل الفصول الدراسية من بيانات الفيديو.
  6. نظام التنبؤ بكثافة الحشود وتوقع نقاط الاختناق قبل حدوثها.
  7. حساب مؤشر سلامة المدرسة (SSI) من خمسة مصادر بيانات مستقلة ومتكاملة.
